你现在接手仓库：fund_agent

目标是将当前 `fund-agent` 渐进式重构为：

```text
由宿主注入 MCP 能力、以 KnowledgeGraph 和 EvidenceGraph 为中枢、
由 AI Planner 主导分析、具备 Critic 反思和 Execution Ledger 输出能力的
AI Financial Research OS。
```

必须遵守以下原则：

1. 以现有仓库为基础渐进重构，不允许推倒重来。
2. 旧流程必须隔离为 legacy adapter，不允许和新主流程混杂。
3. 纯数学计算、风险指标、持仓账本、定投结算必须脱离 LLM、网络、IO。
4. KnowledgeGraph 必须成为 Planner 的上下文层，而不是最终报告附属字段。
5. EvidenceGraph 必须成为最终决策的唯一证据来源。
6. MCP 能力只在 Skill 文档和接口层声明，不允许在 Skill 内硬编码供应商 SDK 或 API 调用。
7. 每个阶段必须新增 pytest，所有改动必须通过测试。
8. 最终投资决策必须满足 `decision-contract.v2`，不能只给方向，必须给执行量、触发条件、失效条件、风险预算和审计轨迹。

# 总体改造目标

当前系统仍然偏向：

```text
CLI analyze
  -> data fetch
  -> quant scoring
  -> news pipeline
  -> report.evidence.json
  -> agent_decisions.json
  -> final report
```

请重构为：

```text
ResearchTask
  -> Planner
  -> KnowledgeGraph query
  -> Skill plan
  -> Tool execution
  -> EvidenceGraph compile
  -> Critic review
  -> DecisionContract v2
  -> ExecutionLedger
  -> FinalThesis
```

旧流程可以继续保留作为对照组，但新主路径必须独立存在。
