# Evidence Contract v2

## EvidenceItem Schema

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `evidence_id` | `str` | Yes | Unique identifier |
| `evidence_type` | `EvidenceType` | Yes | `HardEvidence`, `SoftEvidence`, or `HybridEvidence` |
| `source_type` | `str` | Yes | Origin: `quant_tool`, `news_source`, `sentiment_analysis`, `kg_query`, `llm_inference`, or `hybrid` |
| `timestamp` | `datetime` | Yes | When the evidence was generated |
| `related_entities` | `list[str]` | Yes | Fund/stock/industry codes this evidence pertains to |
| `claim` | `str` | Yes | Human-readable claim |
| `value` | `Any` | Yes | Payload (numeric, dict, list) |
| `confidence_weight` | `float` | Yes | 0.0 to 1.0. HardEvidence = 1.0 always |
| `direction` | `Direction` | Yes | `positive`, `negative`, or `neutral` |
| `version` | `str` | Yes | `"evidence-contract.v2"` |
| `provenance` | `dict` | No | Tracing metadata |

## HardEvidence Rules

1. **Confidence weight is always 1.0** — HardEvidence comes from pure computation, not inference. Enforced at construction via `__post_init__`.
2. **Generated by pure tool functions** — Quant metrics, ledger calculations, KG queries. No IO, no network, no LLM.
3. **No mutable external state** — Must be reproducible. Same inputs always produce the same evidence.
4. **Use `EvidenceItem.from_tool_output()` factory** — Automatically sets `evidence_type="HardEvidence"` and `confidence_weight=1.0`.

## SoftEvidence Rules

1. **Confidence clamped to [0.1, 0.9]** — Always some uncertainty. Defaults to 0.5 if not specified.
2. **Generated from news, sentiment, LLM inference** — Anything with external or model-based uncertainty.
3. **Requires source_type** — No anonymous SoftEvidence. Must trace back to a specific pipeline or model.
4. **Use `EvidenceItem.from_news()` factory** — Automatically sets `evidence_type="SoftEvidence"` and clamps confidence.

## HybridEvidence Rules

1. **Upgraded from SoftEvidence** — Requires 2+ corroborating supporting evidence IDs in the graph.
2. **Confidence boosted by 30% (capped at 0.95)** — Multi-source corroboration increases confidence.
3. **Detected via `EvidenceGraph.upgrade_to_hybrid()`** — Returns the upgraded item or None if conditions aren't met.

## Required Fields Validation

| Field | Validation |
|-------|------------|
| `source_type` | Cannot be empty string |
| `timestamp` | Must be `datetime` instance |
| `related_entities` | Must have at least 1 entity |
| `confidence_weight` | Must be in [0.0, 1.0] |

## Rejection Rules

- Empty `related_entities` → `ValueError`
- HardEvidence with `confidence_weight != 1.0` → `ValueError`
- Missing `source_type` → `ValueError`
- `confidence_weight` outside [0.0, 1.0] → `ValueError`

## EvidenceGraph Schema

`src/schemas/evidence_graph.py`

| Field | Type | Description |
|-------|------|-------------|
| `items` | `dict[str, EvidenceItem]` | Indexed by evidence_id |
| `edges` | `list[tuple[str, str]]` | Relationships between evidence items |

### Edge Types

| Type | Description |
|------|-------------|
| `supports` | One evidence supports another |
| `contradicts` | Opposite directions on shared entities |
| `derived_from` | Evidence was derived from another item |
| `about_entity` | Evidence pertains to a specific entity |

### Operations

| Method | Description |
|--------|-------------|
| `add(item)` | Add evidence (overwrite by ID) |
| `get(evidence_id)` | Retrieve by ID, or None |
| `add_edge(from_id, to_id)` | Link two evidence items |
| `deduplicate()` | Remove near-duplicates (claim similarity > 0.85 AND same entity set) |
| `detect_conflicts()` | Find opposite-direction pairs on shared entities |
| `upgrade_to_hybrid(soft_id, supporting_ids)` | SoftEvidence → HybridEvidence with 2+ corroborating sources |
| `aggregate_confidence(entity)` | Weighted-average confidence per entity |
| `by_entity(entity)` | Filter by related entity |
| `by_type(evidence_type)` | Filter by evidence type |
| `validate()` | Integrity checks (HardEvidence confidence, range validity) |
| `to_dict()` | Serialization with stats summary |
| `hard_evidence_count()` | Count of HardEvidence items |
| `soft_evidence_count()` | Count of SoftEvidence items |
| `hybrid_evidence_count()` | Count of HybridEvidence items |
