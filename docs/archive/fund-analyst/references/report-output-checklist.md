# 输出自检清单

生成完整报告后，必须逐项验证以下清单。当前系统以结构化引擎为主，Agent 负责解释、归因和有限 overlay。

## 硬性完成条件

- [ ] `grep "AGENT_FILL\|<!-- AGENT:" report.md` 返回空。
- [ ] 报告没有 `X%`、`¥XXX`、`[待填]`、`N/A` 被误当成结论使用。
- [ ] 所有仓位、调仓金额、趋势方向、触发条件均来自结构化证据层输出，未被 Agent 手算覆盖。
- [ ] 末尾保留完整风险提示，不出现保证收益、稳赚不赔等违规承诺。

## 逐基金检查

- [ ] 输出数据完整度 A/B/C/D，并解释 `score_confidence`。
- [ ] 完整度 C/D 或 `score_confidence < 0.7` 时，没有主动加仓、重仓买入或加速定投结论。
- [ ] 评分表包含宏观/中观/微观三项；若中观因完整度不足跳过，已说明缺失口径。
- [ ] “量化评分拆解”至少展示关键 `factor_matrix` 因子：因子名、数值、因子分、权重、来源。
- [ ] 微观诊断引用了可用的 Sortino、Sharpe、Jensen Alpha、IR、Beta、最大回撤、波动率或其缺失策略。
- [ ] `trend_matrix` 的短期/中期方向、置信度、驱动项已被解释，正文没有与趋势矩阵相反。
- [ ] `operation_advice` 的动作、目标仓位、调整金额、触发条件已被解释，未被覆盖。
- [ ] 止盈线和止损线使用证据层数字，且带单位；若为估算，已标注 `[数据缺失-已估算]`。
- [ ] QDII 基金额外标注净值延迟、pending、汇率和海外休市风险。

## 新闻前置与催化检查

- [ ] 新闻模块已经在评分前运行；诊断中说明新闻如何影响评分或趋势。
- [ ] 低相关新闻没有被拿来支撑买入或加仓结论。
- [ ] 新闻分析区分旧情绪均值与新催化评价，优先使用 `news_evaluation`、`catalyst_news`、`brief`。
- [ ] 新闻穿透链条包含“事件 -> 重仓/行业链 -> 趋势矩阵 -> 操作建议”。
- [ ] 若新闻为空或接口失败，明确说明趋势/评分仅依据净值和持仓数据，降低置信度。

## 组合风险检查

- [ ] 展示或解释 `portfolio_risk_matrix.cluster_exposures`。
- [ ] 高相关配对（相关系数绝对值 >0.85）已指出重复敞口及处理方式。
- [ ] 成长制造、海外、商品、防守、红利、宽基等暴露簇拥挤时，推荐和调仓解释服从组合风险预算。
- [ ] pending 资金、QDII T+2、定投状态已经进入再平衡解释。
- [ ] 资金分配表中的目标占比、调整金额、操作来自结构化结果；净调仓金额不需要由 Agent 再手算。

## 推荐模块检查

- [ ] 推荐候选说明 `portfolio_role`、`entry_plan`、`risk_budget_impact`。
- [ ] 当前组合缺少防守资产时，解释防守/红利/固收候选为何提升排序。
- [ ] 成长制造暴露过高时，没有继续把半导体、新能源、AI 等同簇候选作为无条件新增。
- [ ] 高相似候选只作为替代观察，不作为叠加买入。
- [ ] 推荐基金代码、名称、主题、风险均来自候选数据，不编造。

## 交易日/非交易日工作流检查

- [ ] 交易日报告重点解释当日盈亏、当日归因、新闻线索和 QDII/pending 状态。
- [ ] 非交易日报告重点解释一周收益贡献、风险暴露、定投质量和本周归因。
- [ ] 交易日和非交易日两套体系没有混用标题或结论。

## 逻辑一致性检查

- [ ] 高分但趋势下行时，结论是持有/等待/分批，不是无条件追高。
- [ ] 低分或低置信度但趋势短期上行时，结论是观察/小额试仓，不是重仓买入。
- [ ] 同簇高相关基金不会同时加速定投，除非给出替代或对冲理由。
- [ ] Agent overlay 如偏离规则建议，已写明证据、幅度、风险和复核触发条件。

如任一项未通过，修正后重新验证全部清单。
