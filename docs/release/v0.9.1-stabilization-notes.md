# v0.9.1 Stabilization Notes

## Investigation: 2 AkShare env failures from v0.9.0

### Failure identification

The v0.9.0 readiness checklist recorded: "3177 passed, 2 pre-existing akshare env failures, 5 skipped".

Reproducing on the current environment (Python 3.13, Windows, no akshare installed):

```
3173 passed, 11 skipped in 202.69s
```

**0 failures** in the current environment. The 2 AkShare failures were environment-specific.

### Root cause analysis

The 2 failures were most likely caused by one of:

1. **akshare installed but offline**: If `akshare` was installed but the machine had no network, `_ensure_akshare()` would succeed (the import works), but `fund_open_fund_info_em()` would fail with a network error. The adapter catches this and returns an error result, but a test asserting `result.ok is True` on a live call would fail.

2. **akshare installed with incompatible version**: If `akshare` was installed but had a different API signature (e.g., different column names), the `UNEXPECTED_COLUMNS` path would trigger, causing assertion failures in tests that expected success.

3. **Environment variable interference**: Some tests may have been affected by environment variables set on the developer machine.

### Key finding

All existing adapter unit tests in `tests/host_data/test_akshare_adapter.py` are **already fully mocked** — they inject `MagicMock` for `akshare` and never touch the real package or network. The failures were NOT in these tests.

The failures likely occurred in:
- `tests/architecture/test_provider_boundaries.py::test_examples_host_data_adapters_may_import_provider_sdks` — this test reads `akshare_adapter.py` source and checks for `"import akshare"` in the text. It does not import akshare itself, so this is unlikely to fail.
- Or a test that was running in an environment where akshare was partially available.

### Fix strategy

1. **Add `live_provider` / `adapter_live` marked tests** — explicit opt-in tests that require real akshare/network. These are skipped by default via `FUND_AGENT_RUN_LIVE_PROVIDER_TESTS` env var.

2. **Ensure all default adapter tests remain mocked** — no changes needed; they already are.

3. **Add explicit `live_provider` skipif pattern** — new tests in `tests/host_data/test_live_provider_adapters.py` use `@pytest.mark.live_provider` and `@pytest.mark.skipif(not os.getenv("FUND_AGENT_RUN_LIVE_PROVIDER_TESTS"))`.

4. **Update pyproject.toml markers** — added `adapter_live` marker.

### Why the fix preserves provider boundary

- Live provider tests are opt-in and skipped by default.
- Core runtime tests remain no-network.
- Adapter unit tests remain fully mocked.
- No real provider SDKs are imported in default test runs.

### Why default tests remain no-network

- All adapter unit tests use `MagicMock` for provider SDKs.
- Live provider tests are behind `FUND_AGENT_RUN_LIVE_PROVIDER_TESTS` env var.
- Architecture boundary tests scan source files, not imports.
- Provider smoke runner (`provider_smoke.py`) is outside core runtime and opt-in.
