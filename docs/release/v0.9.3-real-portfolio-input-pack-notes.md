# v0.9.3 Real Portfolio Input Pack Notes

## Portfolio Input Schemas

- `schemas/fund_portfolio_input.schema.json` — main portfolio input schema
- `schemas/transaction_history.schema.json` — transaction history schema
- `schemas/risk_profile.schema.json` — investor risk profile schema
- `schemas/investment_constraints.schema.json` — investment constraints schema

## Templates

All templates are in `examples/user_portfolio_templates/`:
- `fund_portfolio_input_template.json` — empty template
- `fund_portfolio_input_demo.json` — synthetic demo (3 funds)
- `provider_data_snapshot_template.json` — empty snapshot template
- `provider_data_snapshot_demo.json` — synthetic demo snapshot
- `transaction_history_template.csv` — CSV template
- `risk_profile_template.yaml` — risk profile template
- `investment_constraints_template.yaml` — constraints template

## Portfolio Input Bridge

`src/skills_runtime/workflow/portfolio_input_bridge.py` provides:
- `bridge_portfolio_input()` — converts portfolio input to fund_analysis payload
- Handles missing cost basis, missing snapshots, data quality warnings
- Never fetches live data, never executes trades

## Private Data Handling

- `.gitignore` updated with `local_data/`, `private_data/`, `local_reports/`, `*.private.*`
- `docs/development/private-data-handling.md` provides full guidance
- All demo data is synthetic

## No Real Private Data Committed

All data in templates and demos is synthetic. Real portfolio data must be stored in `local_data/` only.
