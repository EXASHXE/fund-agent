# v0.10.0 Release Notes Draft

## Beta Candidate Scope

v0.10.0 is the first beta candidate for host-integrated personal mutual fund advisory workflows.

## New Since v0.9.0

### v0.9.2 — Provider Adapter Usability
- Provider adapter usability hardening (AkShare, Eastmoney, Xueqiu)
- ProviderResult factory methods: network_error, unexpected_schema, not_implemented, error
- Provider snapshot schema (schemas/provider_data_snapshot.schema.json)
- Provider snapshot templates and synthetic demo
- Live smoke documentation clarified
- Default provider tests are no-network

### v0.9.3 — Real Portfolio Input Pack
- Portfolio input schemas (fund_portfolio_input, transaction_history, risk_profile, investment_constraints)
- Portfolio input templates and synthetic demo
- Provider snapshot template and demo
- Portfolio input bridge (bridge_portfolio_input)
- Private data handling docs and gitignore protections

### v0.9.4 — Markdown / zh-CN Report Export
- Deterministic markdown zh-CN report renderer (render_advisory_report_markdown)
- CLI report output support (render-report subcommand)
- Synthetic demo markdown report output
- 10 required zh-CN report sections
- No LLM generation

### v0.9.5 — Agent Integration Polish
- Agent integration docs (6 flow documents)
- Prompt snippets (5 zh-CN prompt templates)
- Integration examples (4 synthetic examples)
- Forbidden behaviors checklist
- Doc consistency tests

## Known Limitations

- No live data fetching in core runtime
- No autonomous trading
- No broker execution
- Eastmoney/Xueqiu adapters are prototypes
- AkShare adapter coverage is partial
- Not v1.0.0 stable API

## How to Test with Private Data Safely

1. Copy templates to local_data/
2. Fill in real data
3. Run fund-agent analyze-portfolio --input local_data/portfolio.json
4. Output to local_reports/
5. Sanitize before sharing
6. Never commit local_data/ or local_reports/
