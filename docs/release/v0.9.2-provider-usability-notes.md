# v0.9.2 Provider Adapter Usability Notes

## Supported Prototype Capabilities

### AkShare
- FUND_NAV_HISTORY: implemented (live, requires akshare package)
- FUND_PROFILE: NOT_IMPLEMENTED
- FUND_HOLDINGS: NOT_IMPLEMENTED
- INDEX_HISTORY: NOT_IMPLEMENTED
- STOCK_HISTORY: NOT_IMPLEMENTED
- No credentials required

### Eastmoney
- All capabilities: NOT_IMPLEMENTED (endpoint map defined)
- Disabled by default
- Cookie requirements unknown until live smoke

### Xueqiu
- All capabilities: NOT_IMPLEMENTED
- Disabled by default
- Likely requires cookie/token

## Live Smoke Policy

- Provider smoke is opt-in
- Provider smoke may call network
- Provider smoke is outside core runtime
- Default pytest does not run live provider smoke
- Run with: `FUND_AGENT_RUN_LIVE_PROVIDER_TESTS=1`

## Default Test Policy

- Default pytest: no network, no provider SDKs, no API keys
- Live provider tests: skipped by default
- Mocked adapter tests: always run

## Known Limitations

- AkShare adapter coverage is partial
- Eastmoney/Xueqiu are prototype adapters, not production connectors
- No live data fetching in core runtime
- Provider snapshot schema is a contract, not a fetcher

## How Provider Snapshots Feed Later Workflow

1. Host collects live data via adapters
2. Host builds provider_data_snapshot JSON
3. fund-agent consumes snapshot for analysis
4. Snapshot provides NAV history, profiles, holdings, fees, etc.
5. Missing snapshot data is explicitly represented, not fabricated
