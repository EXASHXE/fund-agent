# v0.9.2 Provider Adapter Usability Notes

## Supported Prototype Capabilities

### AkShare
- FUND_NAV_HISTORY: implemented path (requires akshare package installed and network access)
- FUND_PROFILE: NOT_IMPLEMENTED
- FUND_HOLDINGS: NOT_IMPLEMENTED
- INDEX_HISTORY: NOT_IMPLEMENTED
- STOCK_HISTORY: NOT_IMPLEMENTED
- No credentials required for basic endpoints
- **Only NAV history is currently implemented; all other capabilities return NOT_IMPLEMENTED**

### Eastmoney
- All capabilities: NOT_IMPLEMENTED (endpoint map defined as scaffold)
- Disabled by default
- Cookie requirements unknown until live smoke
- **This is a prototype scaffold, not a production connector. Host must implement HTTP calls or run live smoke to verify.**

### Xueqiu
- All capabilities: NOT_IMPLEMENTED
- Disabled by default
- Likely requires cookie/token
- **This is a prototype scaffold, not a production connector. Host must implement HTTP calls or run live smoke to verify.**

## Provider Snapshot Is the Recommended Path

For real testing with actual data, the **provider data snapshot** flow is recommended:
1. Host collects live data via adapter
2. Host builds `provider_data_snapshot` JSON
3. fund-agent consumes the snapshot deterministically
4. No live provider is required inside core runtime

See `schemas/provider_data_snapshot.schema.json` and `examples/user_portfolio_templates/`.

## Live Smoke Policy

- Provider smoke is opt-in
- Provider smoke may call network
- Provider smoke is outside core runtime
- Default pytest does not run live provider smoke
- Run with: `FUND_AGENT_RUN_LIVE_PROVIDER_TESTS=1`

## Default Test Policy

- Default pytest: no network, no provider SDKs, no API keys
- Live provider tests: skipped by default
- Mocked adapter tests: always run
- **No live provider is required for default tests**

## Known Limitations

- AkShare adapter currently has only NAV history implemented; other capabilities are NOT_IMPLEMENTED
- Eastmoney/Xueqiu are prototype scaffolds, not production connectors
- No live data fetching in core runtime
- Provider snapshot schema is a contract, not a fetcher

## How Provider Snapshots Feed Later Workflow

1. Host collects live data via adapters
2. Host builds provider_data_snapshot JSON
3. fund-agent consumes snapshot for analysis
4. Snapshot provides NAV history, profiles, holdings, fees, etc.
5. Missing snapshot data is explicitly represented, not fabricated
